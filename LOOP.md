# Loop Definition — therapy skill

## Cadence

This is a one-shot decision loop, not a recurring pipeline.
Run once per high-stakes decision. Do not re-run unless new information
materializes or the user's situation changes materially.

## Gates

### Kill Gates (any FAIL → HOLD, decision deferred)

- util_trajectory: User cannot name 3 concrete near-term projects enabled by this decision
- emergency_fund: Post-decision reserve < 6 months essential expenses
- payment_mode: High-interest EMI or unaffordable monthly obligation
- hard_gate_any: Structural/spec failure (wrong specs, wrong price, cannot deliver)

### Watch Gates (flagged but not blocking)

- battery_health: Warranty shields worst case, but inspect at handover
- social_comparison: Guilt from others' opinions, confirmed non-signaling motive
- schema_persistence: Doubt remains after all layers resolved → recommends professional support

## Conditions

- STRONG_BUY: All kill gates pass, all layers RESOLVED
- BUY: All kill gates pass, one CONDITIONAL soft gate, clear path to resolve
- HOLD: Any kill gate fails OR any Layer 3/4 probe UNRESOLVED without resolution path

## Exit Criteria

Decision is finalized when:
1. All kill gates PASS
2. All 14 bias probes RESOLVED
3. All 10 schema probes RESOLVED
4. User can state the decision without physical anxiety response (informal check)

## Re-entry

Loop can be re-entered if:
- Core situation changes (income, family, health)
- Structural facts change (price drops, warranty terms discovered, new competitor)
- At least 30 days have passed since last run (prevents rapid re-looping from persistent doubt)
