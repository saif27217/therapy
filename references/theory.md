# Decision-Quality Loop — Theory Reference
## Therapy Skill — Cognitive Bias & Schema Reference

Compiled from the first-person session (July 2026) and distilled into
reusable theory for future decision-quality checks.

---

## 1. The Problem This Framework Solves

Most decision-making frameworks (including standard Loop Engineering L1) are
factual: they check specs, price, capability, and structural risk. They work
perfectly when the blocker is missing information.

They fail when the blocker is the decider. When someone has all the facts,
has done the math, and the verdict is clear — and still can't act — the failure
is not informational. It's psychological:

- Core beliefs about self-worth override rational analysis
- Schema-driven risk inflation amplifies manageable risks into existential threats
- Emotional override (fear, guilt, shame) hijacks the decision system
- Past trauma imports into present situations that don't match

This framework adds two layers (Bias Mitigation + Schema Challenge) on top of
standard factual gate analysis to catch these failures.

## 2. Layer 3 — Bias Mitigation (7 Distortions)

Full question sets in `templates/BIAS-MITIGATION.md`.

| Distortion | Core Mechanism | Key Question |
|------------|---------------|-------------|
| Confirmation Bias | Seeks confirming evidence, ignores contradictions | "What's the strongest argument AGAINST?" |
| Overconfidence | Trusts own logic beyond its validity | "What if I'm wrong in 12 months?" |
| Anxiety/Stress | Emotional pressure clouds judgment | "What's the break-even number?" |
| Loss Aversion | Fear of regret distorts risk | "Which regret hurts more: buying or not?" |
| Social Pressure | Others' expectations undermine confidence | "Would you buy if no one knew?" |
| Complexity Paralysis | Too many variables → can't decide | "If you kept ONE thing, what would it be?" |
| Biological Variation | Irreducible uncertainty feels like risk | "Can you live with the worst case?" |

**Detection rule:** If any answer shows hedging, deflection, self-contradiction, or
deferral → the bias is active. Loop reopens.

## 3. Layer 4 — Schema Challenge (5 Patterns)

Full question sets in `templates/SCHEMA-CHALLENGE.md`.

### 3.1 Emotional Override
A core belief (e.g., "I don't deserve this") triggers anxiety that hijacks
the decision system. The brain prioritizes emotional safety over logical evidence.

**Test:** Is the feeling protecting you from real harm, or from a *belief* about harm?

### 3.2 Cognitive Dissonance
When the proposed action conflicts with a self-concept (e.g., "I'm careful with
money"), the mind creates discomfort and may dismiss evidence to restore consistency.

**Test:** What core belief does this conflict with? What do you gain by proving it wrong?

### 3.3 Overgeneralization
One failure scenario → whole-life cascade. The schema amplifies worst-case into
existential threat. One bad event is treated as predictor of total life failure.

**Test:** What's the *minimum* realistic harm? Is that survivable?

### 3.4 Schema Rigidity (Lack of Permeability)
Old patterns resist new evidence because they feel familiar. A single positive
data point should update the belief, but doesn't. The schema is a known quantity;
trusting the new evidence is unknown — brain prefers known pain.

**Test:** Name evidence that contradicts the schema. Does it update the belief?

### 3.5 Schema-Driven Risk Inflation
Past harm (financial mistake, trauma) is imported into current situations that
are structurally different. The doubt responds to a ghost, not the present.

**Test:** What safeguards exist NOW that didn't exist THEN?

## 4. Scoring Protocol

| Score | Meaning | Action |
|-------|---------|--------|
| RESOLVED | Answer is clean, decisive, no hedging/deflection | Proceed |
| UNRESOLVED | Answer shows rationalization, deflection, self-contradiction | Reopen loop |

- Any single UNRESOLVED → loop reopens on that specific question
- Partial UNRESOLVED → additional probing questions follow
- All RESOLVED → proceed with clarity

## 5. Key Insight from First Session

The decisive question in the therapy layer is often the *utilization* question:

> "Will you actually use what you're buying for the intended purpose within 30 days?"

This single question separates production-tool purchases from signaling purchases.
If utilization is genuine and imminent, schema-based doubt is noise.
If utilization is hypothetical or deferred, the doubt may be correct.

Non-utilization is the most honest failure mode. It cannot be rationalized away.

## 6. When to Recommend Professional Support

After all 4 layers are run, if:
- Schema-driven doubt persists despite clean factual analysis
- The user cannot name a positive precedent that updates their self-trust schema
- The decision has been deferred multiple times with the same doubt pattern
- Core beliefs ("I'm inherently wrong") are generalized beyond this specific decision

Recommend CBT or therapy. The framework is a QC protocol, not treatment.
Persistent schema regardless of evidence quality = referral, not re-analysis.

## 7. Integration with Loop Engineering

```
therapy layer runs AFTER loop-engineering L1 factual analysis.
It is the L2/L3/L4 emotional layer.

Sequencing:
  L1 (hard gates) → only proceeds if factual verdict is ambiguous or user is blocked
  L2 (soft gates) → financial/situational reality
  L3 (bias) → cognitive distortions
  L4 (schema) → core belief patterns
  All clear → proceed
  Any fail → HOLD, not indefinite deferral
```
